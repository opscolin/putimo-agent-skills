# 腾讯云账单对比分析报告生成提示词

## 任务描述

分析指定目录下的腾讯云账单 CSV 文件，对比两个月份的费用数据，生成详细的费用变化分析报告。

## 输入数据说明

### 文件命名规则
- `tencent_billing_YYYYMM_business_summary.csv` - 产品汇总费用文件
- `tencent_billing_YYYYMM_product_summary.csv` - 产品明细费用文件
- `YYYYMM` 格式：202601 代表 2026 年 1 月，202602 代表 2026 年 2 月

### 文件字段说明

**business_summary (产品汇总)**
| 字段 | 说明 |
|------|------|
| 分组 key | 产品编码（如 p_cvm） |
| 分组名称 | 产品名称（如 云服务器 CVM） |
| 原价 | 产品原价 |
| 实际费用 | RealTotalCost，实际代支付费用（**核心关注字段**） |
| 现金费用 | 现金支付费用 |

**product_summary (产品明细)**
| 字段 | 说明 |
|------|------|
| 产品名称 | 产品中文名称 |
| 产品编码 | 产品编码 |
| 子产品名称 | 子产品名称 |
| 子产品编码 | 子产品编码 |
| 资源别名 | 资源别名 |
| 资源 ID | 资源唯一标识 |
| 付费模式 | 按量计费/包年包月 |
| 原价 | 原价 |
| 实际费用 | RealTotalCost，实际代支付费用（**核心关注字段**） |
| 现金支付 | 现金支付金额 |
| 折扣 | 折扣率 |
| 账单类型 | 账单类型 |

## 分析要求

### 1. 总费用计算

```
总实际费用 = Σ(所有产品的实际费用) + 月度计费精度差异调整项
```

**注意**: 
- 必须包含 `billVirtualId` (月度计费精度差异) 行的数据
- 只关注 `RealTotalCost` (实际费用/现金支付) 字段
- 不要使用单个产品费用作为总费用

### 2. 产品费用变化分析

对 business_summary 中每个产品计算：
- 变化金额 = 2 月费用 - 1 月费用
- 变化比例 = (变化金额 / 1 月费用) × 100%

### 3. 识别关键变化

**费用降低 TOP 5**: 按变化金额绝对值排序，取前 5 个下降最多的产品

**费用增长产品**: 列出所有费用增长的产品

**费用持平产品**: 变化比例在 ±0.01% 以内的产品

**特殊变化**: 
- 新增产品（1 月无费用，2 月有费用）
- 消失产品（1 月有费用，2 月无费用）

### 4. 明细原因分析

对费用变化较大的产品（变化比例 > 20% 或 变化金额 > ¥1000），需要：
1. 在 product_summary 中筛选对应产品编码的数据
2. 按子产品/资源 ID 汇总费用
3. 识别具体是哪个子产品或资源导致的费用变化
4. 分析变化原因（使用量变化、规格调整、天数因素等）

### 5. 输出报告结构

```markdown
# 腾讯云账单分析报告 - YYYY 年 M 月 vs N 月

## 一、整体费用概览

| 月份 | 总实际费用 |
|------|-----------|
| 月份 1 | ¥XXX,XXX.XX |
| 月份 2 | ¥XXX,XXX.XX |
| 变化 | -¥XX,XXX.XX (-X.XX%) |

## 二、产品费用变化分析

### 2.1 费用变化汇总表
（完整列出所有产品的费用及变化）

### 2.2 费用降低 TOP 5
（按变化金额排序）

### 2.3 费用增长产品
（列出所有增长的产品）

### 2.4 费用持平产品
（变化比例接近 0 的产品）

## 三、费用变化原因分析

对每个重点变化产品进行详细分析：
- 产品名 (变化比例%, 变化金额)
- 明细变化表格（子产品/资源维度）
- 原因分析

## 四、总结

### 4.1 费用下降主要驱动因素
| 因素 | 影响金额 | 占比 | 说明 |

### 4.2 费用稳定/增长项
| 产品 | 状态 | 说明 |

### 4.3 建议
（基于分析结果给出优化建议）

## 五、费用结构分析（可选）

### 5.1 TOP 5 成本产品对比
### 5.2 费用类型分布（包年包月 vs 按量计费）
```

## 输出要求

1. **数据准确性**: 
   - 所有费用数据保留 2 位小数
   - 百分比保留 2 位小数
   - 总费用必须与原始数据累加结果一致

2. **分析深度**:
   - 不仅列出变化数据，更要分析变化原因
   - 基于 product_summary 的明细数据支撑结论

3. **格式规范**:
   - 使用 Markdown 格式
   - 表格对齐清晰
   - 金额使用 ¥ 符号

4. **建议可操作**:
   - 针对费用异常变化给出具体核查建议
   - 针对成本优化给出可行建议

## 注意事项

1. **不要编造数据**: 所有分析必须基于提供的 CSV 文件内容
2. **不要发散**: 聚焦于账单数据本身的分析
3. **校验总额**: 生成报告前必须校验总费用计算正确
4. **包含调整项**: 月度计费精度差异必须计入总费用
5. **关注 RealTotalCost**: 只分析实际代支付费用，不分析原价

## 示例命令

```bash
# 计算 1 月总费用（包含精度调整）
awk -F',' 'NR>1 {sum+=$4} END {printf "%.2f\n", sum}' tencent_billing_202601_business_summary.csv

# 提取某产品明细
awk -F',' '$2=="p_cdn" {print}' tencent_billing_202601_product_summary.csv
```
